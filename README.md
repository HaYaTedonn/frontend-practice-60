# HTML / CSS / JavaScript 学習リポジトリ

ようこそ！ここは HTML / CSS / JavaScript を**問題形式**で学ぶ場所です。
各問題フォルダの `README.md` に問題文が書かれています。スターターコードを編集して正解を目指してください。

## 📂 問題の構成

| レベル | 問題数 | 難易度 | 内容 |
| --- | --- | --- | --- |
| [01_初級](./01_初級) | 20問 | ★1〜★5 | HTML と CSS の基礎（タグ、色、レイアウト、Flexbox、Grid など） |
| [02_中級](./02_中級) | 20問 | ★3〜★7 | 1ページ制作、レスポンシブ、JavaScript 入門 |
| [03_上級](./03_上級) | 20問 | ★5〜★10 | 実用的な機能の実装（LocalStorage、Fetch API、ゲームなど） |

難易度は **★（最大10）** で表しています。数字が大きい問題ほど難易度が高いので、**番号順に進める**のがおすすめです。

## 🚀 進め方

1. **このリポジトリを fork する**
   自分の GitHub アカウントにコピーを作ります。

2. **fork したリポジトリを clone する**
   ```bash
   git clone https://github.com/あなたのユーザー名/このリポジトリ名.git
   cd このリポジトリ名
   ```

3. **問題を解く**
   - `01_初級/問題01_HTMLの基本構造/` のように、番号順に問題フォルダを開く
   - そのフォルダの `README.md`（問題文）を読む
   - `index.html` / `style.css` / `script.js` を編集して要件を満たす
   - ブラウザで `index.html` を開いて動作確認

4. **ブランチを切ってコミット**
   ```bash
   git checkout -b 初級-問題01
   git add .
   git commit -m "初級 問題01: HTMLの基本構造"
   git push origin 初級-問題01
   ```

5. **プルリクエストを作成**
   GitHub 上で僕（リポジトリオーナー）宛に PR を作ってください。
   タイトル例: `[初級-問題01] HTMLの基本構造`

6. **レビュー → 合格なら Close**
   - 正解 → `正解` ラベル付きで **Close**（🎉 次の問題へ進みましょう）
   - 要修正 → `要修正` ラベル付きでコメント（追加コミットを同じブランチに push して再提出）

   > ⚠️ **合格 PR も merge しません。** main のスターターコードを守るため、**すべての PR は Close で終わる**運用です。
   > あなたの解答は**あなたの fork に永久保存**されるので、個人のアーカイブとして活用できます。

## 🔀 なぜ merge しないの？

もし PR を merge すると、**main の問題ファイルがあなたの解答で上書き**されてしまいます。
次にこのリポジトリを fork した人が、前の人の解答から始めることになるので、スターターコードは常に元のまま保ちます。

| アクション | main の問題ファイル |
| --- | --- |
| Merge | ❌ あなたの解答で上書きされる |
| **Close** | ✅ 元のスターターのまま保たれる |

PR は「**提出 & レビューの場**」として使い、コードの取り込みはしません。

## 📝 プルリクエストのルール

- **1問ごとに 1 PR** が基本（まとめて出さない）
- 編集するのは**その問題フォルダの中のみ**（他の問題や README には触らない）
- コミットメッセージに問題番号を入れる
- **PR タイトルにも問題番号** を付ける（例: `[初級-問題01] HTMLの基本構造`）

## 🌐 ライブプレビュー（GitHub Pages）

このリポジトリは **GitHub Pages** に対応しています。リポジトリオーナーが有効化すると、`https://ユーザー名.github.io/リポジトリ名/` で全問題の一覧ページが開けます。

### 有効化手順（リポジトリオーナー向け）
1. リポジトリの **Settings → Pages**
2. **Source** を「Deploy from a branch」にする
3. **Branch** を `main` / root に設定して Save
4. 数分後、上記 URL で公開される
